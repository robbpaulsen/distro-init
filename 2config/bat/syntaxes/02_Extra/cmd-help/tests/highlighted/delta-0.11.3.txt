[38;2;248;248;242mGIT CONFIG[0m
[38;2;248;248;242m----------[0m

[38;2;248;248;242mBy default, delta takes settings from a section named "delta" in git config files, if one is[0m
[38;2;248;248;242mpresent. The git config file to use for delta options will usually be ~/.gitconfig, but delta[0m
[38;2;248;248;242mfollows the rules given in https://git-scm.com/docs/git-config#FILES. Most delta options can be[0m
[38;2;248;248;242mgiven in a git config file, using the usual option names but without the initial '--'. An example[0m
[38;2;248;248;242mis[0m

[38;2;248;248;242m[delta][0m
[38;2;248;248;242m    line-numbers = true[0m
[38;2;248;248;242m    zero-style = dim syntax[0m

[38;2;248;248;242mFEATURES[0m
[38;2;248;248;242m--------[0m
[38;2;248;248;242mA feature is a named collection of delta options in git config. An example is:[0m

[38;2;248;248;242m[delta "my-delta-feature"][0m
[38;2;248;248;242m    syntax-theme = Dracula[0m
[38;2;248;248;242m    plus-style = bold syntax "#002800"[0m

[38;2;248;248;242mTo activate those options, you would use:[0m

[38;2;248;248;242mdelta --features my-delta-feature[0m

[38;2;248;248;242mA feature name may not contain whitespace. You can activate multiple features:[0m

[38;2;248;248;242m[delta][0m
[38;2;248;248;242m    features = my-highlight-styles-colors-feature my-line-number-styles-feature[0m

[38;2;248;248;242mIf more than one feature sets the same option, the last one wins.[0m

[38;2;248;248;242mSTYLES[0m
[38;2;248;248;242m------[0m

[38;2;248;248;242mAll options that have a name like --*-style work the same way. It is very similar to how[0m
[38;2;248;248;242mcolors/styles are specified in a gitconfig file:[0m
[38;2;248;248;242mhttps://git-scm.com/docs/git-config#Documentation/git-config.txt-color[0m

[38;2;248;248;242mHere is an example:[0m

[38;2;248;248;242m--minus-style 'red bold ul "#ffeeee"'[0m

[38;2;248;248;242mThat means: For removed lines, set the foreground (text) color to 'red', make it bold and[0m
[38;2;248;248;242m            underlined, and set the background color to '#ffeeee'.[0m

[38;2;248;248;242mSee the COLORS section below for how to specify a color. In addition to real colors, there are 4[0m
[38;2;248;248;242mspecial color names: 'auto', 'normal', 'raw', and 'syntax'.[0m

[38;2;248;248;242mHere is an example of using special color names together with a single attribute:[0m

[38;2;248;248;242m--minus-style 'syntax bold auto'[0m

[38;2;248;248;242mThat means: For removed lines, syntax-highlight the text, and make it bold, and do whatever delta[0m
[38;2;248;248;242m            normally does for the background.[0m

[38;2;248;248;242mThe available attributes are: 'blink', 'bold', 'dim', 'hidden', 'italic', 'reverse', 'strike',[0m
[38;2;248;248;242mand 'ul' (or 'underline').[0m

[38;2;248;248;242mThe attribute 'omit' is supported by commit-style, file-style, and hunk-header-style, meaning to[0m
[38;2;248;248;242mremove the element entirely from the output.[0m

[38;2;248;248;242mA complete description of the style string syntax follows:[0m

[38;2;248;248;242m- If the input that delta is receiving already has colors, and you want delta to output those[0m
[38;2;248;248;242m  colors unchanged, then use the special style string 'raw'. Otherwise, delta will strip any colors[0m
[38;2;248;248;242m  from its input.[0m

[38;2;248;248;242m- A style string consists of 0, 1, or 2 colors, together with an arbitrary number of style[0m
[38;2;248;248;242m  attributes, all separated by spaces.[0m

[38;2;248;248;242m- The first color is the foreground (text) color. The second color is the background color.[0m
[38;2;248;248;242m  Attributes can go in any position.[0m

[38;2;248;248;242m- This means that in order to specify a background color you must also specify a foreground (text)[0m
[38;2;248;248;242m  color.[0m

[38;2;248;248;242m- If you want delta to choose one of the colors automatically, then use the special color 'auto'.[0m
[38;2;248;248;242m  This can be used for both foreground and background.[0m

[38;2;248;248;242m- If you want the foreground/background color to be your terminal's foreground/background color,[0m
[38;2;248;248;242m  then use the special color 'normal'.[0m

[38;2;248;248;242m- If you want the foreground text to be syntax-highlighted according to its language, then use the[0m
[38;2;248;248;242m  special foreground color 'syntax'. This can only be used for the foreground (text).[0m

[38;2;248;248;242m- The minimal style specification is the empty string ''. This means: do not apply any colors or[0m
[38;2;248;248;242m  styling to the element in question.[0m

[38;2;248;248;242mCOLORS[0m
[38;2;248;248;242m------[0m

[38;2;248;248;242mThere are four ways to specify a color (this section applies to foreground and background colors[0m
[38;2;248;248;242mwithin a style string):[0m

[38;2;248;248;242m1. CSS color name[0m

[38;2;248;248;242m   Any of the 140 color names used in CSS: https://www.w3schools.com/colors/colors_groups.asp[0m

[38;2;248;248;242m2. RGB hex code[0m

[38;2;248;248;242m   An example of using an RGB hex code is:[0m
[38;2;248;248;242m   --file-style="#0e7c0e"[0m

[38;2;248;248;242m3. ANSI color name[0m

[38;2;248;248;242m   There are 8 ANSI color names:[0m
[38;2;248;248;242m   black, red, green, yellow, blue, magenta, cyan, white.[0m

[38;2;248;248;242m   In addition, all of them have a bright form:[0m
[38;2;248;248;242m   brightblack, brightred, brightgreen, brightyellow, brightblue, brightmagenta, brightcyan, brightwhite.[0m

[38;2;248;248;242m   An example of using an ANSI color name is:[0m
[38;2;248;248;242m   --file-style="green"[0m

[38;2;248;248;242m   Unlike RGB hex codes, ANSI color names are just names: you can choose the exact color that each[0m
[38;2;248;248;242m   name corresponds to in the settings of your terminal application (the application you use to[0m
[38;2;248;248;242m   enter commands at a shell prompt). This means that if you use ANSI color names, and you change[0m
[38;2;248;248;242m   the color theme used by your terminal, then delta's colors will respond automatically, without[0m
[38;2;248;248;242m   needing to change the delta command line.[0m

[38;2;248;248;242m   "purple" is accepted as a synonym for "magenta". Color names and codes are case-insensitive.[0m

[38;2;248;248;242m4. ANSI color number[0m

[38;2;248;248;242m   An example of using an ANSI color number is:[0m
[38;2;248;248;242m   --file-style=28[0m

[38;2;248;248;242m   There are 256 ANSI color numbers: 0-255. The first 16 are the same as the colors described in[0m
[38;2;248;248;242m   the "ANSI color name" section above. See https://en.wikipedia.org/wiki/ANSI_escape_code#8-bit.[0m
[38;2;248;248;242m   Specifying colors like this is useful if your terminal only supports 256 colors (i.e. doesn't[0m
[38;2;248;248;242m   support 24-bit color).[0m


[38;2;248;248;242mLINE NUMBERS[0m
[38;2;248;248;242m------------[0m

[38;2;248;248;242mTo display line numbers, use --line-numbers.[0m

[38;2;248;248;242mLine numbers are displayed in two columns. Here's what it looks like by default:[0m

[38;2;248;248;242m 1  ⋮ 1  │ unchanged line[0m
[38;2;248;248;242m 2  ⋮    │ removed line[0m
[38;2;248;248;242m    ⋮ 2  │ added line[0m

[38;2;248;248;242mIn that output, the line numbers for the old (minus) version of the file appear in the left column,[0m
[38;2;248;248;242mand the line numbers for the new (plus) version of the file appear in the right column. In an[0m
[38;2;248;248;242munchanged (zero) line, both columns contain a line number.[0m

[38;2;248;248;242mThe following options allow the line number display to be customized:[0m

[38;2;248;248;242m--line-numbers-left-format:  Change the contents of the left column[0m
[38;2;248;248;242m--line-numbers-right-format: Change the contents of the right column[0m
[38;2;248;248;242m--line-numbers-left-style:   Change the style applied to the left column[0m
[38;2;248;248;242m--line-numbers-right-style:  Change the style applied to the right column[0m
[38;2;248;248;242m--line-numbers-minus-style:  Change the style applied to line numbers in minus lines[0m
[38;2;248;248;242m--line-numbers-zero-style:   Change the style applied to line numbers in unchanged lines[0m
[38;2;248;248;242m--line-numbers-plus-style:   Change the style applied to line numbers in plus lines[0m

[38;2;248;248;242mOptions --line-numbers-left-format and --line-numbers-right-format allow you to change the contents[0m
[38;2;248;248;242mof the line number columns. Their values are arbitrary format strings, which are allowed to contain[0m
[38;2;248;248;242mthe placeholders {nm} for the line number associated with the old version of the file and {np} for[0m
[38;2;248;248;242mthe line number associated with the new version of the file. The placeholders support a subset of[0m
[38;2;248;248;242mthe string formatting syntax documented here: https://doc.rust-lang.org/std/fmt/#formatting-parameters.[0m
[38;2;248;248;242mSpecifically, you can use the alignment and width syntax.[0m

[38;2;248;248;242mFor example, the default value of --line-numbers-left-format is '{nm:^4}⋮'. This means that the[0m
[38;2;248;248;242mleft column should display the minus line number (nm), center-aligned, padded with spaces to a[0m
[38;2;248;248;242mwidth of 4 characters, followed by a unicode dividing-line character (⋮).[0m

[38;2;248;248;242mSimilarly, the default value of --line-numbers-right-format is '{np:^4}│'. This means that the[0m
[38;2;248;248;242mright column should display the plus line number (np), center-aligned, padded with spaces to a[0m
[38;2;248;248;242mwidth of 4 characters, followed by a unicode dividing-line character (│).[0m

[38;2;248;248;242mUse '<' for left-align, '^' for center-align, and '>' for right-align.[0m

[38;2;248;248;242mhttps://github.com/dandavison/delta/issues.[0m
